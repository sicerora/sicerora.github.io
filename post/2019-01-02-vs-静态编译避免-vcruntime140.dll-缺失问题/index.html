VS 静态编译避免 vcruntime140.dll 缺失问题

| jni

VS 编译 dll 时默认引入动态链接的多线程库（/MD），而动态链接会有外部依赖，如果项目比较简单，为了方便在新环境使用，可以改为使用静态链接的多线程库（/MT）来避免外部依赖。

操作步骤

  1. “项目 -> 配置属性 -> C/C++ -> 代码生成” 设置为 “多线程（/MT）”
  2. 编译方式改为 Release

PS: Debug 编译方式对应“多线程调试（/MTd）”
